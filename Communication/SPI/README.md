# SPI (Serial Peripheral Interface)

A 4-wire protocol that's faster than I2C, commonly used where data throughput matters — SD cards, TFT displays, RFID readers, and wireless modules like the NRF24L01.

## Key Concepts

- **MOSI (Master Out, Slave In)** — data sent from the microcontroller to the device
- **MISO (Master In, Slave Out)** — data sent from the device back to the microcontroller
- **SCK (Serial Clock)** — clock signal generated by the microcontroller, keeps both sides in sync
- **CS / SS (Chip Select / Slave Select)** — unlike I2C's shared-address model, SPI identifies which device to talk to by pulling its dedicated CS pin LOW. Each SPI device needs its own CS pin, even though MOSI/MISO/SCK can be shared across multiple devices.

## Common Modules Using SPI (in this repo)

| Module | Notes |
|---|---|
| RC522 RFID Reader | Needs MOSI, MISO, SCK, plus its own CS and RST pins |
| MicroSD Card Module | Standard SPI, needs its own CS pin |
| NRF24L01 Wireless Module | 2.4GHz transceiver, SPI-based |
| ILI9341 / other TFT Displays | SPI-based, often also needs a DC (data/command) pin |

## Wiring Pattern

```
Board MOSI →  Module MOSI/SDI
Board MISO ←  Module MISO/SDO
Board SCK  →  Module SCK/SCLK
Board CS   →  Module CS/SS  (a unique digital pin per device)
Board VCC  →  Module VCC (check voltage rating — many SPI modules are 3.3V only)
Board GND  →  Module GND
```

On Uno/Nano, hardware SPI pins are fixed (MOSI: 11, MISO: 12, SCK: 13 — note SCK overlaps with the onboard LED pin on Uno/Nano, so the LED may flicker during SPI activity). On Mega, they're on pins 50–52. CS can be any free digital pin you choose in code.
